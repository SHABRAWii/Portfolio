Software lead and developer:

  As a software lead, Planning is one of my responsibilities.

The following charts shows the software architecture of the project. The software is divided into three main parts:

1.Host Machine: Responsible for high-level control, including task planning and path planning. It provides a Webserver Dashboard for operators to monitor the robot's status and control its functions remotely.

2.Raspberry Pi 4 B 8Gb: The heart of the robot's operation, running the ROS (Robot Operating System). The Raspberry Pi handles all control algorithms and hardware interactions. Key features include:

  • Logging System: Captures and stores operational data, aiding in debugging and performance monitoring.
  • SLAM (Simultaneous Localization and Mapping): A critical function that enables the robot to map its surroundings and navigate autonomously within them.
  • Gazebo Simulation: Provides a simulated environment for testing algorithms before deployment, ensuring safety and reliability.

3.External Controller (ESP32): Acts as a bridge between the Raspberry Pi and other hardware components. It manages the communication protocols and synchronization needed for the robot's real-time operations. Noteworthy responsibilities include:
  • Omni Wheel Odometry Module: Responsible for precise calculations of the robot's position and orientation, which are vital for accurate navigation.
  • Obstacle Avoidance: Ensures the robot can safely navigate through its environment, avoiding any obstacles in real-time.
  • Return to Charge Point: Automatically directs the robot back to its charging station when battery levels are low.
Key hardware components such as the Lidar sensor, MPU9250 (9-Axis) IMU, and the Stepper Driver and Motor for the Lifting Mechanism are integrated into this architecture, allowing the robot to perform complex tasks with precision and reliability.

This architecture not only enhances the robot's operational efficiency but also ensures a scalable solution capable of handling various tasks in dynamic environments.

  Setup the OS environment and needed tools and backup certain milestones.

As part of my responsibilities, I installed and prepared an Ubuntu Focal Fossa environment, which served as the backbone for all the development and testing processes. To ensure stability and to safeguard against potential issues, I implemented a robust backup strategy at critical milestones during the setup and configuration phases. This approach not only helped in maintaining a stable development environment but also allowed for quick recovery in case of any unexpected failures.

The choice of Ubuntu Focal Fossa was driven by its LTS (Long Term Support) status, which provides a secure and reliable platform for long-term projects. During the setup, I installed all necessary development tools, configured the environment according to project needs, and ensured that the system was optimized for performance. Regular backups were taken using both system snapshots and external backups to preserve the state of the environment at crucial points in the development lifecycle.

Below are some of the key aspects of the OS environment setup:

  • System Installation: Installed Ubuntu Focal Fossa, selected during the setup process due to its compatibility with ROS and other required software.
  • Tool Configuration: Configured essential tools like Git, Docker, and other development utilities.
  • Backup Strategy: Established a routine backup schedule, using both local and cloud storage solutions to ensure that all critical milestones were securely saved.

This meticulous approach to setting up and maintaining the OS environment not only facilitated smooth development but also provided a safety net, ensuring that work could continue without significant downtime in case of system issues.
